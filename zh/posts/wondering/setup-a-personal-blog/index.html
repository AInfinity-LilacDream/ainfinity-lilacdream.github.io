零 · 前言#

这是我的第一篇博客。因为学着搭建博客的时候踩了太多坑，特此来记录一下，尽可能清晰，也给后来人一个参考。

如标题，博客采用Hugo+PaperMod+github pages搭建。下面我将分步记录搭建过程，以及一些踩过的坑。

在正式开始搭建之前，你需要：

  1. 安装至少一个包管理器。npm, chocolatey等均可。
  2. 拥有一个github账户，在电脑上安装了git。若git clone时出现网络问题，请尝试科学上网。
  3. 拥有一个个人域名，这里推荐腾讯云或阿里云备案，大厂有保障。

一 · Hugo-extended的安装#


Hugo安装指南：https://gohugo.io/installation/windows/

Hugo官方入门指南：https://hugo.opendocs.io/getting-started/

首先，在一个合适的位置新建一个文件夹，用于存放你的博客文件。进入你建好的文件夹，点击上方的路径栏

输入cmd，在当前路径下打开cmd。

Hugo现有三个发行版本：standard, extended, extended/deploy. 官方推荐我们直接安装hugo-extended。此处使用npm命令，其他包管理器同理。

1

npm install hugo-extended

这个命令将会在当前目录下安装hugo-extended。安装完后，会多出一个node_modules目录和package.json文件。这些是必要的依赖包，我们可以不用管。

随后，检查hugo-extended是否正确安装。

出现如图所示版本号则已正确安装。


二 · 建立博客目录 / 应用PaperMod主题#


PaperMod官方Wiki：https://github.com/adityatelange/hugo-PaperMod/wiki

在当前目录新建一个hugo博客目录。

1

hugo new site blog --format yaml

进入刚创建的文件夹：

1

cd blog

初始化git仓库：

1

git init

随后，使用github submodule安装PaperMod主题：

1
2

git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
git submodule update --init --recursive

至此，你已经初始化好了PaperMod主题。你可以通过下列命令预览并调试你的网站：

1

hugo server

在PaperMod刚安装好时，网站是空白的。这是因为，PaperMod是一个高度定制化的主题，我们需要进行个性化配置，才能让网站显示出内容。下面将进行网站主题的配置。

4. 部署标签页#


以下为参考文档的原文摘录：

  • PaperMod主题是支持多语言的，但是标签页不支持，需要修改对应的 html 模板。
  • 主题提供的语言支持文件再themes/PaperMod/il8n里面，其中zh.yaml即简体中文。
  • 为了防止更新主题时还原修改过的文件，需要在网站根目录进行修改，这样可以覆盖主题原有文件的效果。

4.1 覆盖默认翻译#

在博客根目录下新建i18n/文件夹，复制themes/PaperMod/i18n/zh.yaml到i18n/zh.yaml。

四 · 发布网站#


本博客采用github pages托管，并利用CNAME部署到个人域名上，从而使得博客能够被公网访问。

参考文档：保姆级教程：从零构建GitHub Pages静态网站


1. 将博客托管至Github Pages#


本部分需要你拥有Github账号。如无，请至Github官网注册。部分大陆地区裸连Github可能会有些慢，你或许需要科学上网。

首先，创建一个新的repository，需要设置为public。建议选择不创建.readme文件。

你的仓库需要命名为yourname.github.io。

随后，进入仓库settings，在Build and deployment一栏中，确保Source为Deploy from a branch，Branch选择默认的master。这是为了你的push能够自动部署。

在博客根目录打开CMD，输入hugo。该命令将会将你的博客编译为静态网站。

随后，进入新生成的public文件夹，该文件夹存放了可用于部署的网站。

1

cd public

初始化git仓库：

1

git init

添加远程库：

1

git remote add origin your-repository-name

新建commit：

1
2
3

git add .
git commit -m "feat: impl blog pages"
git push --set-upstream origin master

随后打开你的repository，等待一到两分钟，应该可以发现Github已经自动部署该网站。你应当可以通过在浏览器中访问your-name.github.io来看到你的网站！


2. 将博客部署到个人域名#


本部分需要你拥有至少一个个人域名。如没有，你应当购买一个。购买审批流程一般在两到三天左右。

一份教程：腾讯云域名注册及解析全方位指南

在博客的public目录下，创建CNAME文件。在CNAME文件中写入你的域名名称，如https://ainfinitythecoder.cn。在当前路径打开CMD，随后新建commit提交更改。

1
2
3

git add .
git commit -m "chore: add CNAME from local"
git push origin

此时，等待一到两分钟，Github Pages应该已经自动将网站部署到你的个人域名下。
